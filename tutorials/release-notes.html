6.0.0

2018年9月27日にリリース

このリリースには、ColumnSortingプラグインに対する変更のみが含まれています。変更に関する詳細な情報はプルリクエスト#5411の説明と私たちのドキュメントにあります。

大きな変更
  • Handsontable Pro用の新しいMultiColumnSortingプラグインとシームレスに連携するように、ColumnSortingプラグインの一部をリファクタリングして書き直しました。これにより、以前のバージョンのプラグインが抱えていた複数の問題を解決しました。
    また、後方互換性のない変更がいくつか導入されました。
    • プラグインの設定項目（sortIndicator、sortEmptyCells、sortFunctionなど）は、プラグインの設定範囲に移動されました。
    • プラグインの初期化設定は、プラグイン設定のinitialConfigプロパティに格納されています。
    • sortFunction設定項目はcompareFunctionFactoryに変わり、比較関数を返すファクトリに変換されました（そして前述のようにプラグインの設定範囲に移動されました）。
    • sortIndicator設定項目はindicatorに名前が変更されました（そして前述のようにプラグインの設定範囲に移動されました）。
    • 比較関数の構造が変更されました // これまで： function numericSort(sortOrder, columnMeta) { return function ([, value], [, nextValue]) { // これから： function numericSort(sortOrder, columnMeta) { return function (value, nextValue) {
    • sortメソッドの引数が再編成されたため、ソート設定をオブジェクトとして受け入れます。 hot.getPlugin('columnSorting').sort({ column: 0, sortOrder: 'asc' });
    • いくつかのパブリックメソッドが書き直され、名前が変更され、プライベートメソッドに設定されました。
      • getNextOrderState
      • loadSortingState
      • saveSortingState
    • beforeColumnSortおよびafterColumnSortフックは、異なる一連の引数を受け取ります。 詳しくは、私たちの資料を確認してください。
    • columnSortingを定義してupdateSettingsメソッドを呼び出すと、新しいソート構成が設定されます。
変更点

上記の重大な変更とは別に、このリリースではソートプラグインにいくつかの追加の変更が加えられています。

  • 新しいプラグイン - MultiColumnSortingを追加しました。テーブルをソートするときに複数の列を使用できます。
    CEのColumnSortingと同様に機能しますが、複数の新しい機能が導入されています。この新機能に関する詳細な情報は資料とプルリクエスト#101の説明にあります。
  • 現在の並び変える時の標識を矢印の形に置き換えました。
  • ソート標識はデフォルトで表示されます。
  • headerActionオプションを使用して、ヘッダーをクリックしてソートの動作を無効にする機能を追加しました。

対応するHandsontable CEのバージョンは6.0.0です。

New API
  • hot.getSelectedLast()は最後に選択されたレイヤーの座標（[row, col, rowEnd, colEnd]）を持つ配列を返します。この変更が行われる前はhot.getSelected()として動作します。
  • hot.getSelectedRangeLast()はテーブルに適用された最後の選択座標を含むCellRangeオブジェクトを返します。この変更が行われる前はhot.getSelectedRange()として動作します。
  • hot.alter('remove_row', [[1, 4], [10, 1]])は連続していない行の削除をサポートします。行インデックスを渡す代わりに、最初の項目が行のインデックスで、2番目の項目が削除される行の数である配列の配列を渡すことができます。
  • hot.alter('remove_col', [[1, 4], [10, 1]])は列以外上記と同じです。その新機能をサポートするのは、「削除」アクションだけです。
後方互換性のない変更
  • hot.getSelected()はすべてのレイヤーの座標（[[row, col, rowEnd, colEnd], [row, col, rowEnd, colEnd] ...]）を持つ配列の配列を返します。
  • hot.getSelectedRange()はすべてのレイヤーの座標（[{CellRange}, {CellRange} ...]）を持つCellRangeオブジェクトの配列を返します。
  • これまでhot.selection.empty()はでしたがこれからはhot.emptySelectedCells()になります
  • 選択色の変更：
    - 境界, #89aff9 -> #4b89ff
    - 背景, #b5d1ff -> #005eff
    - 現在の選択中の境界, #5292f7 -> #4b89ff
  • multiSelect設定を削除し、selectionMode: 'single'に置き換えました。
  • 新しいselectionModeオプションを追加しました。'single'（以前はmultiSelect: false）、'range'（以前はmultiSelect: true）、または'multiple'（新しい非連続モード）のいずれかに設定できます。
後方互換性のある変更
  • afterSelection
    - これまで: afterSelection(row, column, rowEnd, columnEnd, preventScrolling)
    - これから: afterSelection(row, column, rowEnd, columnEnd, preventScrolling, selectionLayerLevel)
  • afterSelectionByProp
    - これまで: afterSelectionByProp(row, prop, rowEnd, propEnd, preventScrolling)
    - これから: afterSelectionByProp(row, prop, rowEnd, propEnd, preventScrolling, selectionLayerLevel)
  • afterSelectionEnd
    - これまで: afterSelectionEnd(row, column, rowEnd, columnEnd)
    - これから: afterSelectionEnd(row, column, rowEnd, columnEnd, selectionLayerLevel)
  • afterSelectionEndByProp
    - これまで: afterSelectionEndByProp(row, prop, rowEnd, propEnd)
    - これから: afterSelectionEndByProp(row, prop, rowEnd, propEnd, selectionLayerLevel)

上記のすべてのフックに対してselectionLayerLevel引数を追加しました。selectionLayerLevelは、どの選択レイヤーが現在変更されているかを示す番号です。最初の選択では、この値は0になり、新しく追加されたレイヤーの数が増えます。
