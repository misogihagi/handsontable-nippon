このプロジェクトへの参加は大歓迎です。バグを修正したり、新しい機能を提案したい場合は、 新しいプルリクエストを送ることができますが、最初に次の一般的なルールに従っていることを確認してください。

  1. Contributor License Agreementに署名して、コードへの変更を公開することを許可します。
  2. 別のブランチで変更を加えます。これはマージプロセスをスピードアップします。
  3. あなたのプルリクエストを常に `master`ではなく` Develop`ブランチにしてください。
  4. `dist/`のファイルを編集しないでください（例： `handsontable.js`、` handsontable.css`、 `handsontable.full.js`、` handsontable.full.css`、`languages/all.js`） および`languages/`（例：`en-US.js`、`pl-PL.js`）ディレクトリ。代わりに、`src/`ディレクトリ内のファイルを編集してから`npm run build`を使ってビルドしてください。 より多くの情報は ここで見つけることができます。
  5. 重要：変更した場合は、`test/e2e/`（エンドトゥーエンドテスト用）、`test/unit/`または`src/3rdparty/walkontable/test/spec/`に少なくとも1つのテストケースを追加してください。 それは私たちが問題を理解し、恒久措置であることの確認を助けるでしょう。 私たちのテストプロセスとは
  6. コードをnpm run lintを実行してリントしてください。私たちのコーディングスタイルはAirbnb JavaScript Styleをもとにづいています。
  7. すべての変更に詳細な説明を追加してください。
