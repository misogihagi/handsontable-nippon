概要

国際化は、Handsontableが特定の言語に翻訳するためにUIのテキストを変更することを簡単にしてくれます。 私たちは開発者に定義済みの言語（言語セットをロードして設定を変更することで適用できます）と、 既存の言語ファイルのテンプレートを使用して作成された独自の言語セットを使用する機能を提供します。

私たちはHandsontableに新しい翻訳を追加するのを手伝ってくれる寄稿者を探しています。もっと詳しく

準備された言語ファイルをロードする

国際化機能を正しく使用するには、言語セットを読み込む必要があります。重要なのは、それらがHandsontableファイルの後に含まれているということです。 必要なファイルを入手することでそれを行うことができます（UMD標準を使用して作成されています）。

  1. HTMLファイル内の/dist/languagesフォルダからロード

    <script type="text/javascript" src="dist/handsontable.full.js"></script>
<script type="text/javascript" src="dist/languages/all.js"></script>

    注：/dist/languagesフォルダにあるファイルの圧縮バージョンを使用することもできます。

  2. プロジェクトのルートの下のモジュールシステムで使用される/languagesフォルダーからロード 以下の例はECMAScript 6でそれを使う方法を示しています（バンドラーかTypeScriptが必要）。

    Handsontable Pro Editionの場合: 

    import Handsontable from 'handsontable-pro';
import 'handsontable-pro/languages/pl-PL';

    Handsontable Community Editionの場合: 

    import Handsontable from 'handsontable';
import 'handsontable/languages/pl-PL';

このように含まれている言語はファイルをロードした直後に使用する準備ができています。 Handsontableの他の言語固有のファイルおよび言語登録についての詳細は、このセクションにあります。

翻訳されたコンテキストメニューを表示するには、セルを右クリックしてください。Handsontableをロードした後に言語ファイルがロードされました。

国際化に関連するすべての主要ファイルの階層

この段落内の情報は、主に新しい言語を追加したり、既存の言語を編集したい人々にとって有用です。 用意された言語を使用する簡単な方法はここに記述されています。 以下で言及しているすべての変更はHandsontable CEです。

Handsontable CEでは、現在、言語ファイルとして扱うことができるファイルを含む3つのフォルダーがあります。

  • /dist/languages*
  • /languages*
  • /src/i18n/languages

最初の2つのフォルダからのファイルは、webpackによってUMDモジュールとしてビルドされます（次のnpmスクリプトを使って準備することができます：build:languagesとbuild:languages.min）。最後の/srcディレクトリには、ES2015規格を使用して作成されたモジュールが含まれています。

言語辞書を使用するには、言語セットを登録する必要があります。登録とは、Handsontable.languages.registerLanguageDictionaryメソッドを使用してHandsontable用に辞書をグローバルにロードするプロセスです。

これらのディレクトリの一般的な目的は、以下の表にまとめられています。

名称 /dist/languagesおよび/languagesフォルダー内のファイル /src/i18n/languagesフォルダー内のファイル
使い方 別ファイルとして格納 Handsontable内部でライブラリの一部として使用（デフォルトの言語）
ファイルがどこから作られるか ビルドプロセス中にwebpackによって作成され、Babelによって変換 手動で作成され、Babelによって変換されることができる
言語登録 webpackのビルドプロセス中に適用 自動的に登録されない
修正 自動的に作成されるので、編集するべきではない 作成も編集も手動

/dist/languagesフォルダ内のファイルは、/languages内にあるファイルのコピーですが、2つの違いがあります。 前者のフォルダには、追加のファイルの圧縮バージョンが含まれています。 また、index.jsファイルは含まれていません（/languagesフォルダには、すべての言語を簡単にロードするために含まれています）。

これら両方のフォルダーには、すべての言語辞書を含むall.jsファイルが含まれています。

カスタム言語を作成する

Handsontableに簡単に適用できるよう実装用のカスタム言語セットを作成したり、それらを共有したりすることができます。

言語ファイル

私たちのライブラリの成長するにあたってコミュニティが不可欠であるということは、私たちにとって本当に重要です。私たちは翻訳して共有することを応援しています!

追加の言語ファイルは、選択した言語コードに対応する名前でHandsontable CEリポジトリのsrc/i18n/languagesフォルダーに配置する必要があります（下記参照。例：es-VE.js）。私たちにプルリクエストを送信することであなたの翻訳をHandsontableライブラリに組み込むことができます。重要なのは、変更が/languagesおよび/dist/languagesディレクトリに行われていないことです。私たちのリリースマスターは、ビルドプロセスでそこに置かれるファイルを生成します。その後、< code>Handsontable Proの言語を使用できるようになります。

この段落の下部に、サンプル言語の完全なテンプレートがあります。私たちは デフォルトの言語パックに基づくようにしています。ファイル作成プロセスの一部を以下に説明します

  1. ファイルは、authors（カンマで区切り、例：Authors: Chris Wick, John Kyle）、"last updated" date（形式：mmm dd, yyyy、例：Last updated: Jan 01, 2017）とdescriptionを含むコメントで始まる必要があります。 

    /**
* @preserve
* Authors: Chris Wick, John Kyle
* Last updated: Nov 15, 2017
*
* Description: Definition file for Spanish - Venezuela language-country.
*/

  2. その後、翻訳に使用する辞書キーをインポートする必要があります。

    import * as C from '../constants';

  3. 言語辞書オブジェクトには、languageCodeキー（2つの小文字、ハイフン、2つの大文字。例：languageCode: 'es-PY'）を含める必要があります。languageCodeキーはHandsontableの設定と翻訳に使用する辞書キーで使うlanguageプロパティで使用される言語コードを決定します。 

    const dictionary = {
  languageCode: 'es-VE',
  [C.CONTEXTMENU_ITEMS_ROW_ABOVE]: 'Insertar fila arriba',
  ...
}

  4. 最後に、作成した辞書をデフォルトの出力先に配置します。

    export default dictionary;

  5. 簡単なサンプル言語の辞書は、以下の引用のようになります。/languagesおよび/dist/languagesフォルダーはビルドプロセスによって生成されます。このセクションに示すように、これらのローカライズ版のファイルを含めることができます。ロードした後、Handsontableの言語設定をes-VEに変更することによってにその言語を使うことができます。 

    /**
* @preserve
* Authors: Chris Wick, John Kyle
* Last updated: Nov 15, 2017
*
* Description: Definition file for Spanish - Venezuela language-country.
*/
import * as C from '../constants';

const dictionary = {
  languageCode: 'es-VE',
  [C.CONTEXTMENU_ITEMS_ROW_ABOVE]: 'Insertar fila arriba',
};

export default dictionary;

  6. src/i18n/languages/index.jsファイル内にすでに作成されているファイルをインポートして、下に示されているようにエクスポートします（キーはアルファベット順）。

     import deCH from './de-CH';
 import deDE from './de-DE';
 import enUS from './en-US';
+import esVE from './es-VE';
 import plPL from './pl-PL';

 export {
   deCH,
   deDE,
   enUS,
+  esVE,
   plPL
 };

  7. 東西！これで使用したり、他の人と共有できる言語を作成しました。作成されたプルリクエストを受け入れるためにユーザーからの少なくとも5つのポジティブなフィードバックを待ちます。

ローカル言語

Handsontableパッケージの一部ではない言語辞書を登録することができます。そのためには、次のセクションで簡単に説明されているHandsontable.languages.registerLanguageDictionary静的メソッドと静的定数Handsontable.languages.dictionaryKeysを使用します。 

const C = Handsontable.languages.dictionaryKeys;

Handsontable.languages.registerLanguageDictionary({
  languageCode: 'morse',
  // モールス信号に変換します
  [C.FILTERS_BUTTONS_OK]: '--- -•-'
});

翻訳でカスタムキーを使用する

カスタムキーを含む言語辞書を登録できます。これらのエントリは他のキーと同じように使用できます。 そのため、事前定義済みの定数（src/i18n/constants.jsファイル内にあり、Handsontable.languages.dictionaryKeysエイリアスからアクセスできるもの）を使用することに限定されません。 

const enUSDictionary = Handsontable.languages.getLanguageDictionary('en-US');

enUSDictionary.customKey = 'Hello world';

Handsontable.languages.registerLanguageDictionary(enUSDictionary); // 再登録
Handsontable.languages.getTranslatedPhrase('en-US', 'customKey'); // 'Hello world'

静的なHandsontableメソッドとプロパティ

Handsontableには、言語に関連するいくつかの静的メソッドとプロパティがあります。これらはグローバルHandsontable変数のlanguagesキーに格納されています。

Handsontable.languages.getLanguageDictionary(languageCode: String)

特定の言語コードの言語辞書を取得します。

返り値：Object

Handsontable.languages.getLanguagesDictionaries()

登録されている言語辞書を取得します。

返り値：Array

Handsontable.languages.getTranslatedPhrase(languageCode: String, dictionaryKey: String, extraArguments: Mixed)

指定された辞書キーのフレーズを取得します。

返り値：Object

Handsontable.languages.registerLanguageDictionary(languageCodeOrDictionary: Mixed, dictionary: Object)

特定の言語コードの言語辞書を登録します。その後、それを使用することが可能になります。

返り値：Object

Handsontable.languages.dictionaryKeys

辞書の定数です。

含み：Object
