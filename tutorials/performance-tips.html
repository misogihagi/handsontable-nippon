Handsontableはグリッドを正しく表示するために複数の計算を実行します。 最も要求されるアクションは、ロード、変更、およびスクロールイベントの実行です。 各操作はパフォーマンスを低下させますが、それらのほとんどは避けられません。

パフォーマンスラボを使用して、さまざまな構成での実行時間を測定しましょう。 いくつかのテストでは、アプリケーションのパフォーマンスを向上させる可能性がある方法があることを示しました。 これらは特定のケースでしか動作しませんが、それらがあなたのアプリにもうまく適用できることを願っています。

一定サイズに設定

テーブルの列に一定のサイズを設定してみてください。 こうすると、Handsontableは各列に最適な幅を計算する必要はありません。 これを行うには、Handsontableインスタンスの設定のcolWidthsプロパティで列幅を定義します。次に例を示します。 

var hot = new Handsontable(obj, {
    // その他のオプション
    colWidths: [50, 150, 45]
});

詳しくは、私たちの資料を参照してください。

Handsontableは列幅の計算をしなくなるので、 テーブルの内容が提供された幅で列の内側に収まるようにする必要があります。

autoRowSizeとautoColumnSizeのどちらかあるいはその両方を無効にする

autoRowSizeおよびautoColumnSizeオプションの値を変更することができます。 それらはあなたがテーブルの初期化の間になされる幅/高さ関連の計算の量を定義することを可能にします。

詳細については、行と列についての私たちの文書を参照してください。

事前レンダリングされる行と列の数を定義する

テーブルの表示部分の外側にレンダリングする行数と列数を明示的に指定できます。 より少ない数を設定することで（より少ない要素がレンダリングされるので） より良い結果を得ることができますが、時にはより大きな数を設定したほうが うまくいくかもしれません（各スクロールイベントで行われる操作が少なくなるので）。 これらの設定を微調整して芯を食えば、実装したHandsontableの感覚が向上する可能性があります。

詳細については、行と列についての私たちの文書を参照してください。

経験則：スタイルを多用しない

背景やフォントの色などを変更してもパフォーマンスが低下することはありませんが、 CSSアニメーション、トランジション、およびその他の計算を費やす属性を追加しすぎると、 パフォーマンスに影響が出る可能性があります。スタイルは計画的に。
