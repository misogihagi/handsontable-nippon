ブラウザ上のテスト

ブラウザでJasmineのテストを実行するには、次のページにアクセスしてください。

  • <YOUR_LOCALHOST_ADDRESS>/test/E2ERunner.html - Handsontableのエンドツーエンドのテストスイート（Handsontable ProとCEの両方で利用可能）
  • <YOUR_LOCALHOST_ADDRESS>/test/MobileRunner.html - Handsontableのモバイルテストスイート（Handsontable CEでのみ利用可能）
  • <YOUR_LOCALHOST_ADDRESS>/src/3rdparty/walkontable/test/SpecRunner.html - Walkontableのテストスイート（Handsontable CEでのみ利用可能）

NPMによるテスト（CLI）

コマンドラインで（Puppeteerを使用して）Jasmineのテストを実行するには、まずnpm installを実行して必要なすべての依存関係をインストールします。 NPMが必要なモジュールの取り出しを終えたら、テストの準備は完了です。

Handsontable Proで利用可能なコマンド：

  • npm test - すべてのテストケース（Handsontable PROとHandsontable CE）を実行します。
  • npm test:e2e - Handsontable PROのエンドツーエンドテストのみを実行します。
  • npm test:unit - Handsontable PROユニットテストのみを実行します。
  • npm test:handsontable - Handsontable CEのエンドツーエンドテストのみを実行します。
  • npm run test:e2e --testPathPattern=selection - ファイル名"selection"に一致するエンドツーエンドのテストとスイートのみを実行します。
  • npm run test:unit --testPathPattern=array - ファイル名"array"に一致するエンドツーエンドのテストとスイートのみを実行します。
  • npm run test:e2e.dump - test/E2ERunner.htmlファイルを生成します。これはブラウザ環境（Chrome、PhantomJSなど）で実行され、テストに合格したかどうかを確認できます。
  • npm run test:e2e.dump -- --watch - テストファイルで検出されたすべての変更についてE2ERunner.htmlファイルを生成します。デバッグの企てに役立ちます。

Handsontable CEで利用可能なコマンド：

  • npm test - すべてのテストケース（Handsontable CEとWalkontable）を実行します。
  • npm test:e2e - Handsontableのエンドツーエンドテストのみを実行します。
  • npm test:unit - Handsontableのユニットテストのみを実行します。
  • npm test:walkontable - runs only Walkontable tests.
  • npm run test:e2e --testPathPattern=selection - ファイル名"selection"に一致するエンドツーエンドのテストとスイートのみを実行します。
  • npm run test:unit --testPathPattern=array - ファイル名"array"に一致するエンドツーエンドのテストとスイートのみを実行します。
  • npm run test:e2e.dump - test/E2ERunner.htmlファイルを生成します。これはブラウザ環境（Chrome、PhantomJSなど）で実行され、テストに合格したかどうかを確認できます。
  • npm run test:e2e.dump -- --watch - テストファイルで検出されたすべての変更についてE2ERunner.htmlファイルを生成します。 デバッグの企てに役立ちます。
  • npm run test:mobile.dump - MobileRunner.htmlを生成します。これはモバイルデバイスのブラウザ環境で実行され、テストに合格したかどうかを確認します。
