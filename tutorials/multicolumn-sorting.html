概要

列ソートプラグインは、データソースとHandsontableレンダリングモジュールの間のプロキシとして機能します。これは表示されている行のインデックス（「視覚のインデックス」と呼ばれる）を、データソース内の対応する行のインデックス（「物理インデックス」と呼ばれる）にマップしたり、その逆を行うことができます。これにより、データソースの内部構造を変更せずに、ユーザーに表示されている行の順序を変更できます。ソート操作は、使用するブラウザーやソートするデータセットのサイズに関係なく、安定したソートアルゴリズムを使用して実行されます。

columnSortingプラグインとmultiColumnSortingプラグインの両方を同時に有効にしないでください。

基本的なプラグインの設定

プラグインを有効にする最も簡単な方法は、multiColumnSortingキーをtrueに設定することです。以下の例に示すように、APIメソッドを使用してヘッダーをクリックしてソートすることができます。ctrl/cmdキーを押しながら複数の列見出しをクリックすると、クリックされた列がソートスタックに追加され、その結果、定義された順序で列がソートされます。さらに、デフォルトでは：

  • 最初に列はソートされません
  • ソート標識は有効になります
  • 空のセルはソートされません
  • sortメソッドはデフォルトのcompare関数を使います（それらについての詳細はこちらをご覧ください）。

拡張的なプラグイン設定

objectとして設定を提供することでプラグインオプションをカスタマイズできます。設定可能なオプションは次のとおりです。

  • initialConfig (Array), 一部の列の初期ソートステータスを決定します。以下のキーを含むオブジェクトのリスト
    • column (Number), ソートされた列の視覚的なインデックスを決定します
    • sortOrder (String), 列をソートする順序を決定します（取り得る値は'asc'と'desc'）
  • indicator (Boolean), 並べ替え順序の標識を表示するかどうかを定義します（並べ替え順序を指定する列ヘッダーの矢印アイコン）
  • sortEmptyCells (Boolean), 空のセルをソート処理に含むかどうかを定義します
  • headerAction (Boolean), ヘッダーをクリックしてテーブルをソートするかどうかを定義します
  • compareFunctionFactory (Function), 比較関数ファクトリを定義します（このセクションで詳しく説明します）

デフォルトの比較機能（さまざまな種類のデータのソート）

ネイティブのArray.sortメソッドと同様に、私たちの内部ソートアルゴリズムはcompare関数（コンパレータとも呼ばれます）を使用します。異なる種類のセル（日付、数値、テキストなど）は異なる方法で処理されます。それぞれがそれぞれのデータ型をソートするための独自のコンパレータを持っています。

その結果、さまざまな種類のデータが正しくソートされていることがわかります。下記の例でわかるように、Handsontableは単に列の宣言されたデータ型を必要とします。

カスタムの比較機能

ソートアルゴリズムに独自の比較機能を提供することができます。この関数は、ネイティブのArray.sortメソッドの引数と同じように見えます（ここで説明を読んでください）。compareFunctionFactoryが設定で定義されているときにプラグインによって処理されます。我々がと呼ぶものは、比較関数ファクトリと呼ぶものは、このキーの下に置かれなければなりません。ファクトリはパラメータsortOrderとcolumnMetaを受け取り、比較関数を返します。

カスタムソートの実装

プラグインはsetSortConfigメソッドを公開しています。これは内部のソート状態を設定する目的で設計されています。このメソッドではデータセットはソートされないことに注意してください。ただし、isSortedメソッドとgetSortConfigメソッドは、事前に設定された設定に対応する結果を返します。カスタムソートを使用するには、次の操作を行います。

  • falseを返すbeforeColumnSortフックのコールバックを作成します
  • データをソートします（例えば、サーバーサイドで）
  • setSortConfigメソッドを使用してソートの内部状態を設定します

下記の記事のカスタムソートの実装を見てください。 

beforeColumnSort: function(currentSortConfig, destinationSortConfigs) {
  const columnSortPlugin = this.getPlugin('multiColumnSorting');

  columnSortPlugin.setSortConfig(destinationSortConfigs);

  // const newData = ... // 計算されたデータセット。つまりAJAX呼び出しから。

  // this.loadData(newData); // 新しいデータセットをロードします。

  return false; // デフォルトのソートアクションの禁止。
}

プラグインフック

プラグインには2つのフックがあります。

  • beforeColumnSortは、名前が示すとおり、ソートの前に実行されます。つまり、コールバック内でgetSortConfigメソッドによって取得された実際のソート設定は、フック呼び出しの前に保持されていたソート設定と同じになります。beforeColumnSortのコールバックはfalseを返し、テーブルのソートを中止します。その結果、afterColumnSortフックは呼び出されません。
  • afterColumnSortは、beforeColumnSortフックのコールバックがfalseを返さない限り、常にソート後に実行されます。

clearSortメソッドを使用するとき、または処理されない設定にしたときにもフックが実行されることに注意してください（有効性の確認は失敗します）。

プラグインAPI

プラグインによって公開されるメソッドのリスト：

  • getSortConfig
  • isSorted
